Dashboard Overview

Overview

During my summer internship, I partnered with leadership to design interactive dashboards and visuals that translated complex customer feedback data into clear, actionable insights for enterprise teams.

FeedbackNow is a B2B SaaS platform that aggregates customer feedback from multiple channels—surveys, support tickets, reviews, social media—into a unified analytics dashboard. The company had grown rapidly, but their existing interface struggled to scale with increasing data complexity and diverse user needs.

My role focused on redesigning the core dashboard experience and developing a component library that could accommodate future feature expansion while maintaining visual consistency and usability.

Business Goal

Reduce time-to-insight for enterprise customers while improving platform scalability and visual consistency.

The Problem

User Pain Points

Through stakeholder interviews and user research sessions, I identified several key challenges:

  • Information overload: Users felt overwhelmed by dense charts and competing metrics
  • Poor hierarchy: Critical insights were buried alongside secondary data
  • Limited customization: Different roles needed different views but the system was rigid
  • Inconsistent patterns: Visual language varied across features, creating confusion

Technical Constraints

Working within a production environment meant balancing design ambition with practical limitations:

  • Existing React codebase requiring backwards compatibility
  • Performance requirements for handling large datasets (100k+ feedback entries)
  • Tight 8-week timeline coinciding with a major product launch
  • Limited engineering resources for implementation

Design Process

Research & Analysis

I began by conducting a comprehensive audit of the existing system and competitive landscape:

01

User Interviews

Spoke with 12 customers across different roles (analysts, managers, executives) to understand their workflows

02

Competitive Analysis

Analyzed enterprise analytics platforms (Tableau, Looker, Amplitude) for patterns and best practices

03

Data Audit

Reviewed actual customer data to understand scale, complexity, and edge cases

04

Technical Review

Collaborated with engineers to understand system architecture and constraints

Key Insights

Research revealed that different user personas had fundamentally different needs:

  • Executives wanted high-level trends and anomalies at a glance
  • Product Managers needed to drill into specific features and time periods
  • Analysts required granular filtering and export capabilities
  • Support Teams prioritized real-time alerts and quick issue identification

Design Solution

Information Architecture

I restructured the dashboard around a progressive disclosure model, prioritizing the most critical information while maintaining access to detail:

  • Summary cards showing KPIs and trends in a scannable format
  • Sentiment visualization using color-coded sentiment analysis
  • Topic clusters automatically grouping similar feedback themes
  • Drill-down panels allowing users to explore specific segments

Visual Design System

Working closely with the lead designer, I developed a comprehensive component library addressing the consistency issues:

Color System

Semantic color palette: positive (green), negative (red), neutral (gray), with accessible contrast ratios

Typography

Clear hierarchy using font weight and size, optimized for dense data displays

Data Visualization

Standardized chart types with consistent styling, tooltips, and interactions

Components

Reusable UI elements: cards, filters, tables, modals—all documented in Figma

Interaction Design

Key interaction patterns focused on efficiency and discoverability:

  • Smart defaults: Dashboard auto-configures based on user role
  • Contextual filtering: Click any metric to filter the entire view
  • Saved views: Users can save and share custom configurations
  • Keyboard shortcuts: Power users can navigate efficiently without mouse

Implementation & Collaboration

Working with Engineering

Close collaboration with the development team was crucial to success. I learned to:

  • Document designs with detailed specs and interactive prototypes
  • Participate in sprint planning and story sizing
  • Conduct QA reviews and provide implementation feedback
  • Make pragmatic trade-offs when technical constraints arose

Stakeholder Management

Regular communication with leadership ensured alignment:

  • Weekly design reviews with CEO and product leadership
  • User testing sessions with select enterprise customers
  • Iterative refinement based on feedback and usage data

Outcomes & Impact

Measurable Results

Post-launch metrics demonstrated significant improvements:

  • 42% reduction in average time to locate specific insights
  • 67% increase in daily active users engaging with dashboard
  • Net Promoter Score improved from 32 to 58 among enterprise customers
  • Support tickets related to UI confusion decreased by 51%

Learnings

Embrace constraints: Working within tight timelines and technical limitations forced me to prioritize ruthlessly. I learned that perfect is the enemy of shipped, and that incremental improvements often matter more than revolutionary redesigns.

Design for scale: Enterprise products need to handle edge cases and diverse user needs. The component system we built ensured consistency as new features were added, proving that systematic thinking pays dividends.

Communication is design: As much time went into explaining decisions as making them. Learning to articulate rationale to engineers, executives, and customers was as important as the visual design itself.

What's Next

The dashboard continues to evolve post-internship. The team is now working on AI-powered insight recommendations and advanced customization features—all built on the foundation of the design system we established.
