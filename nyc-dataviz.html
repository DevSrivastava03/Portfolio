Overview

An ambitious exploration of New York City's urban data through immersive 3D visualization, synthesizing transportation networks, demographic patterns, and infrastructure systems into a cohesive, explorable interface.

This project emerged from a fascination with how cities function as complex adaptive systems. NYC generates massive amounts of data daily—subway ridership, taxi movements, building permits, demographic shifts—but this data typically exists in isolated datasets, difficult to understand holistically.

The challenge was to create a visualization system that could render this complexity accessible and engaging, transforming abstract numbers into spatial understanding while maintaining performance and usability.

Challenge

How might we visualize NYC's urban systems in a way that reveals patterns invisible in traditional 2D maps or data tables?

Research & Discovery

Data Sources

I began by identifying and evaluating multiple NYC open data sources:

  • NYC Open Data Portal: Building footprints, subway stations, demographic census data
  • MTA Real-Time Feeds: Subway and bus movement data
  • Taxi & Limousine Commission: Trip records and traffic patterns
  • Department of City Planning: Zoning maps and land use data

Technical Constraints

Early prototyping revealed several challenges:

  • Rendering 1M+ building polygons in real-time
  • Managing multiple simultaneous data layers without performance degradation
  • Creating intuitive 3D navigation that doesn't disorient users
  • Designing responsive interactions that work across devices

Design Process

Information Architecture

I developed a layered approach to data visualization, allowing users to progressively explore complexity:

01

Base Layer

3D building geometry showing NYC's physical form

02

Data Overlay

Color-coded visualizations of demographic, transit, or economic data

03

Temporal Animation

Time-based changes showing evolution and movement patterns

04

Detail View

Drill-down into specific neighborhoods or data points

Interaction Design

Navigation in 3D space requires careful consideration. I designed a control system that balances exploratory freedom with guided discovery:

  • Orbital camera controls for intuitive rotation and zoom
  • Landmark-based navigation allowing quick jumps to key locations
  • Filter panel for toggling data layers and adjusting visualization parameters
  • Timeline scrubber for temporal data exploration

Technical Implementation

Architecture

The system is built with modern web technologies, prioritizing performance and maintainability:

Three.js

3D rendering engine, custom shaders for data visualization

React

UI framework for controls and data management

D3.js

Data processing and scale generation

WebGL

GPU-accelerated rendering for smooth performance

Performance Optimizations

  • Level-of-detail (LOD) system reducing geometry based on camera distance
  • Frustum culling to only render visible buildings
  • Instanced rendering for repeated geometries
  • Web Workers for data processing off the main thread

Current Status & Next Steps

What's Working

  • Successfully rendering 100,000+ building polygons at 60fps
  • Intuitive camera controls tested with 10+ users
  • Data pipeline processing and visualizing multiple data sources
  • Responsive design working across desktop and tablet

In Development

  • Temporal animations: Showing changes over time (day/night cycles, seasonal patterns)
  • Advanced filtering: More granular control over data visualization
  • Storytelling mode: Guided tours highlighting specific urban phenomena
  • Mobile optimization: Touch controls and performance improvements

Learnings

This project taught me the importance of balancing ambition with pragmatism. Early versions attempted to visualize too much simultaneously, resulting in visual noise and performance issues. Through iteration, I learned to prioritize clarity and progressive disclosure—showing users one layer at a time while making it easy to explore deeper.
